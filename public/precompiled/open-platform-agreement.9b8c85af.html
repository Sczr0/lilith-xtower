1. 引言

欢迎使用由 Phigros Query（空间站「塔弦」） 提供的开放平台（以下简称“开放平台”）。 本协议用于规范开发者在开放平台中的注册、认证、API 调用、密钥管理及相关行为。

当您点击同意、完成登录、创建 API Key、调用任何开放接口，均视为您已阅读并同意本协议全部内容。 如您不同意本协议，请立即停止使用开放平台相关服务。

2. 定义

2.1 开发者

指完成 GitHub 登录并在开放平台中获取开发者身份的自然人、法人或组织。

2.2 API Key

指由开放平台签发的调用凭证，用于识别调用主体、执行权限校验、限流、审计与风险控制。

2.3 Scope

指 API Key 对应的权限范围，例如 public.read、profile.read 等。

2.4 开放接口

指开放平台对外提供的 API 端点，包括但不限于歌曲检索、排行榜查询、存档解析、RKS 历史查询等。

3. 服务内容与边界

3.1 服务性质

开放平台用于向开发者提供标准化数据能力接入，不承诺永久免费、不承诺特定 SLA，不承诺任何业务收益。

3.2 服务调整

开放平台有权基于安全、性能、合规或运营需要，调整接口、字段、配额、限流、鉴权方式与访问策略。

3.3 版本策略

开放接口采用版本化管理。非破坏性变更可在当前版本增量发布；破坏性变更将通过新版本发布。

4. 开发者账户与认证

4.1 账户基础

开发者通过 GitHub OAuth 登录获取开发者会话。开发者应自行保证 GitHub 账户控制权与安全性。

4.2 会话安全

开发者不得以任何方式伪造、劫持、共享或转售开发者会话。若发现异常登录行为，应及时退出并联系维护者。

4.3 账户状态管理

若开发者存在违规、攻击、滥用或法律风险行为，平台有权限制、冻结或终止其开发者账户能力。

5. API Key 使用规范

5.1 密钥发放与显示

API Key 明文仅在创建或轮换时展示一次，开发者需自行安全保存。平台不提供明文找回。

5.2 密钥保管义务

开发者应妥善保管 API Key，不得将密钥暴露在前端公开仓库、客户端安装包、日志、截图或公开文档中。

5.3 密钥轮换与撤销

开发者应定期轮换 API Key。发生泄露风险时，应立即撤销并重新签发新密钥。

5.4 责任承担

在密钥被撤销前，基于该密钥发起的调用行为均视为开发者行为，相关责任由开发者承担。

6. 调用规范与限制

6.1 权限最小化

开发者应按最小权限原则申请并使用 Scope，不得越权调用。

6.2 限流与风控

平台将按 API Key、客户端 IP、接口类型等维度执行限流、熔断与风控。 触发限流或风控时，平台可拒绝调用、临时封禁或永久禁用相关密钥。

6.3 禁止行为

开发者不得从事包括但不限于以下行为：

（1）绕过鉴权、伪造身份、批量撞库、探测漏洞；
（2）利用开放平台发起 DDoS、扫描、恶意抓取；
（3）违反法律法规处理个人信息；
（4）将接口能力用于作弊、外挂、诈骗、侵犯他人权益；
（5）转售、出租、共享或公开传播 API Key。

7. 数据与隐私

7.1 日志与审计

平台可能记录请求元数据与审计事件，包括但不限于请求时间、接口路径、状态码、调用来源、密钥标识、操作记录等。

7.2 数据最小化

平台仅在实现开放能力与安全治理所需范围内处理数据，不承诺保留期限固定不变。

7.3 合规义务

开发者应确保其业务场景、数据处理流程和用户告知机制符合适用法律法规要求。

8. 可用性与免责

8.1 无担保声明

开放平台按“现状”提供，不对连续可用、无中断、无错误、无延迟、完全准确作出保证。

8.2 免责范围

对于因第三方依赖故障、网络波动、基础设施异常、不可抗力、攻击事件导致的中断或损失，平台在法律允许范围内不承担超出法定义务的责任。

8.3 责任上限

在适用法律允许范围内，平台对开发者承担的累计责任以开发者近 12 个月向平台实际支付金额为上限；如无付费记录，则以人民币 0 元为上限。

9. 协议变更与终止

9.1 协议更新

平台有权更新本协议并在合理位置公示。协议更新后，继续使用开放平台即视为同意更新后的条款。

9.2 终止

若开发者违反本协议或触发重大风险，平台有权在不另行通知的情况下暂停或终止全部或部分服务能力。

10. 联系方式

如您对本协议有疑问，或需反馈安全问题、违规线索、数据处理请求，请通过站内“关于”页面提供的联系方式联系维护者。

本协议适用中华人民共和国法律。因本协议引发的争议，双方应先协商解决；协商不成的，提交平台运营主体所在地有管辖权的人民法院处理。
